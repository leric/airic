# process.cancel

Cancel the active process when it no longer fits the conversation.

## When to use

- The user changes direction or rejects the workflow.
- The process assumptions no longer hold.

## Patterns

- Explain why the process is being cancelled.
- Return to normal mode-based thinking after cancel.

## Avoid

- Cancelling without telling the user when they may expect the workflow to continue.
