# process.list

List available processes with summaries and activation policies.

## When to use

- Debugging or when the process index in context is insufficient.
- The user asks what processes exist.

## Patterns

- During normal chat, the Available Processes section in context is usually enough.
- Use list when tool-based discovery is clearer than re-reading context.

## Avoid

- Listing processes on every turn when the index is already injected.
